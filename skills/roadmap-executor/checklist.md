# Checklist

1. [ ] Locate the source of truth for the roadmap (e.g., a `TODO.md`,
       `ROADMAP.md`, or a specific prompt context).
2. [ ] Identify the single, highest-priority uncompleted task.
3. [ ] Formulate an execution plan specifically for this task.
4. [ ] Execute the necessary code changes, tests, and documentation updates.
5. [ ] Verify the task against the project's standards.
6. [ ] Update the roadmap document (mark as done, check the box, or report
       status back to the user).
7. [ ] Prompt the user or system for permission to proceed to the next task.
